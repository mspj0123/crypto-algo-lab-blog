## 背景

暗号資産の自動売買Botを運用する中で、テクニカル分析だけでは偽シグナルを完全に除去できないという課題がありました。

MACD + RSI + EMA200の組み合わせでエントリー条件を絞っていましたが、シグナルの約50%が負け取引になっていました。

そこで、**LightGBM（機械学習）によるフィルター**を追加して勝率向上に取り組みました。

---

## アプローチ

### 取引対象の絞り込み

まず、9ペアから**大型通貨5ペア（BTC/ETH/XRP/SOL/AVAX）**に絞り込みました。

バックテストの結果、小型通貨（ETC/LSK/XEM/IOST）は勝率0%で、対象から除外することで効率が大幅に改善しました。

### MLフィルターの設計

エントリーシグナル発生時に、以下15個の特徴量からLightGBMで「このシグナルが勝ちになる確率」を予測します。

| カテゴリ | 特徴量 |
|---|---|
| モメンタム | RSI(1h), RSI(4h), MACD値, MACDヒスト |
| ボラティリティ | ATR比, BB %B |
| トレンド | EMA200傾き, 価格vs EMA200, EMA50 vs EMA200 |
| モメンタム | 5本リターン, 20本リターン |
| 時間 | 時間帯, 曜日 |
| 強さ | ADX |

スコアが **0.55以上** のシグナルのみエントリーを許可します。

### 学習データの作成

- yfinanceで過去6ヶ月の1時間足データを取得
- 既存のエントリー条件（MACD GC + RSI + EMA200 + 出来高）でシグナルを抽出
- ATRトレイリングストップで実際の勝敗をラベル付け

## バックテスト結果

過去1ヶ月のシミュレーション（初期資金10万円）：

| 条件 | 取引数 | 勝率 | 損益 | PF |
|---|---|---|---|---|
| MLフィルターなし（5ペア） | 多数 | 約40% | -9,053円 | — |
| **MLフィルターあり** | **8件** | **87.5%** | **+8,276円** | **7.98** |

## 実装の工夫

### フォールバック設計

MLモデルが存在しない・ロード失敗の場合は、**全シグナルを通過**させます。モデルなしでも動作が止まらない安全設計です。

### 週次自動再学習

7日ごとに自動でモデルを再学習します。市場環境の変化（概念ドリフト）に対応するためです。

```python
# config.py
ML_RETRAIN_INTERVAL_DAYS = 7   # 7日ごとに再学習
ML_TRAIN_MONTHS          = 6   # 過去6ヶ月データを使用
ML_THRESHOLD             = 0.55  # 勝率予測55%以上のみ通過
```

## まとめ

テクニカル分析だけのBot（勝率40〜50%）に、LightGBMフィルターを追加することで**勝率87.5%・PF7.98**を達成しました。

ただし、これは過去データでの結果です。実運用での検証を続けて効果を確認していきます。
