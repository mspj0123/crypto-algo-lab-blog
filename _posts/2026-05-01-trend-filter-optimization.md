## 背景

HybridBoost戦略では、ロングエントリーの前提条件として「EMA200が上向き」というトレンドフィルターを設定しています。

これまでは **4時間足のEMA200** を使用していましたが、「もっと短い足に変えたらどうなるか？」という疑問から検証を行いました。

---

## バックテスト概要

過去3ヶ月のBTC/JPYデータで、足種ごとの成績を比較しました。

### 検証条件

- 対象期間：過去3ヶ月
- 対象通貨：BTC/JPY
- エントリー条件：MACD GC + RSI≥50 + 出来高フィルター + EMA200トレンドフィルター
- 決済：ATRトレイリングストップ

### 結果比較

| 足種 | 取引数 | 勝率 | 損益 | PF |
|---|---|---|---|---|
| フィルターなし | 多数 | 約46% | — | — |
| 4時間足EMA200 | — | — | +15,975円 | 1.99 |
| **2時間足EMA200** | — | — | **+41,863円** | **3.32** |
| 1時間足EMA200 | — | — | +41,447円 | 2.97 |

---

## 考察

### なぜ2時間足が良かったか

**4時間足の問題点**：
- 足が粗すぎて、短期的なトレンド転換を見逃す
- エントリーチャンスを過剰に絞り込む

**1時間足の問題点**：
- ノイズが多く、一時的な上昇でも「上向き」と判定してしまう
- 偽シグナルが増加する傾向

**2時間足のバランス**：
- トレンドの捉え方が適度（4時間より敏感、1時間よりノイズ少）
- PF3.32は4時間足の1.67倍、1時間足より若干良好

---

## 実装の変更点

`config.py` に `EMA_TREND_TF` を追加し、**1行で足種を切り替えられる設計** にしました。

```python
# config.py
EMA_TREND_TF = "2h"  # "4h" / "2h" / "1h" で切り替え可能
```

```python
# market_data.py
def fetch_trend_ohlcv(self, pair: str = "btc_jpy", limit: int = 300) -> pd.DataFrame:
    tf_map = {"1h": "1hour", "2h": "2hour", "4h": "4hour"}
    period = tf_map.get(config.EMA_TREND_TF, "2hour")
    return self._get_ohlcv_cached(pair=pair, period=period, limit=limit)
```

キャッシュのTTLも足種ごとに最適化：

```python
_OHLCV_TTL = {
    "1hour": 600,   # 10分
    "2hour": 900,   # 15分
    "4hour": 1800,  # 30分
}
```

## まとめ

4時間足EMA200から2時間足EMA200に変更することで、バックテスト上の損益が **+15,975円 → +41,863円（約2.6倍）** に改善しました。

足種の選択はトレード戦略において重要なパラメータであることを改めて確認しました。
